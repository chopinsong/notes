Maven 使用方法与私有仓库配置教程

一、Maven 简介与安装

1.1 Maven 简介

Apache Maven 是一个项目管理和构建自动化工具，主要用于 Java 项目。

主要功能：

  • 项目构建管理
  • 依赖管理
  • 项目信息管理
  • 项目报告生成
  • 多模块项目管理

核心概念：

  • POM (Project Object Model) - 项目对象模型
  • 坐标 (Coordinates) - groupId、artifactId、version
  • 仓库 (Repository) - 本地仓库、中央仓库、私有仓库
  • 生命周期 (Lifecycle) - 构建过程的标准化

1.2 Maven 安装与配置

前置条件：

确保已安装 JDK 8 或更高版本：

java -version

Windows 安装：

  1. 从 Maven 官网 下载二进制包
  2. 解压到目录，如 C:\apache-maven-3.9.0
  3. 配置环境变量：
    • MAVEN_HOME = C:\apache-maven-3.9.0
    • PATH 中添加 %MAVEN_HOME%\bin

Linux/macOS 安装：

# 下载并解压
wget https://archive.apache.org/dist/maven/maven-3/3.9.0/binaries/apache-maven-3.9.0-bin.tar.gz
tar -xzf apache-maven-3.9.0-bin.tar.gz
sudo mv apache-maven-3.9.0 /opt/maven

# 配置环境变量 (添加到 ~/.bashrc 或 ~/.zshrc)
export MAVEN_HOME=/opt/maven
export PATH=$MAVEN_HOME/bin:$PATH

验证安装：

mvn -version

配置文件位置：

  • 全局配置：$MAVEN_HOME/conf/settings.xml
  • 用户配置：~/.m2/settings.xml
  • 本地仓库：~/.m2/repository

二、Maven 项目结构与配置

2.1 标准项目结构

Maven 采用约定优于配置的原则，定义了标准的项目目录结构：

my-project/
├── pom.xml                    # 项目对象模型文件
├── src/
│   ├── main/                  # 主代码目录
│   │   ├── java/             # Java 源代码
│   │   ├── resources/        # 资源文件
│   │   └── webapp/           # Web 应用资源 (Web项目)
│   └── test/                 # 测试代码目录
│       ├── java/             # 测试 Java 代码
│       └── resources/        # 测试资源文件
└── target/                   # 构建输出目录

创建标准项目：

# 创建简单 Java 项目
mvn archetype:generate \
  -DgroupId=com.example \
  -DartifactId=my-app \
  -DarchetypeArtifactId=maven-archetype-quickstart \
  -DinteractiveMode=false

# 创建 Web 项目
mvn archetype:generate \
  -DgroupId=com.example \
  -DartifactId=my-web-app \
  -DarchetypeArtifactId=maven-archetype-webapp \
  -DinteractiveMode=false

2.2 POM.xml 配置详解

POM (Project Object Model) 是 Maven 项目的核心配置文件。

基本 POM 结构：

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    
    <modelVersion>4.0.0</modelVersion>
    
    <!-- 项目坐标 -->
    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0.0</version>
    <packaging>jar</packaging>
    
    <!-- 项目信息 -->
    <name>My Application</name>
    <description>A sample Maven project</description>
    <url>http://www.example.com</url>
    
    <!-- 属性配置 -->
    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <junit.version>5.8.2</junit.version>
    </properties>
    
    <!-- 依赖管理 -->
    <dependencies>
        <!-- 依赖项配置 -->
    </dependencies>
    
    <!-- 构建配置 -->
    <build>
        <!-- 插件配置 -->
    </build>
    
</project>

重要元素说明：

元素 描述 示例
groupId 组织标识符 com.example
artifactId 项目标识符 my-app
version 版本号 1.0.0
packaging 打包类型 jar, war, pom

三、依赖管理

3.1 依赖配置

添加依赖：

<dependencies>
    <!-- Spring Boot Starter -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
        <version>2.7.0</version>
    </dependency>
    
    <!-- JUnit 测试 -->
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter</artifactId>
        <version>${junit.version}</version>
        <scope>test</scope>
    </dependency>
    
    <!-- MySQL 驱动 -->
    <dependency>
        <groupId>mysql</groupId>
        <artifactId>mysql-connector-java</artifactId>
        <version>8.0.33</version>
        <scope>runtime</scope>
    </dependency>
</dependencies>

依赖范围 (Scope)：

范围 描述 示例
compile 编译、测试、运行时都需要 (默认) Spring Framework
test 仅测试时需要 JUnit
runtime 运行和测试时需要，编译时不需要 数据库驱动
provided 编译和测试时需要，运行时由容器提供 Servlet API
system 类似 provided，但需要显式指定 jar 路径 本地 jar 文件

依赖版本管理：

<!-- 使用 dependencyManagement 统一管理版本 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework</groupId>
            <artifactId>spring-framework-bom</artifactId>
            <version>5.3.21</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

排除传递依赖：

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-tomcat</artifactId>
        </exclusion>
    </exclusions>
</dependency>

四、Maven 生命周期与命令

4.1 生命周期

Maven 有三个内置的构建生命周期：

1. Default 生命周期 (主要阶段)：

阶段 描述
validate 验证项目是否正确
compile 编译源代码
test 运行单元测试
package 打包编译后的代码
verify 验证包是否有效
install 安装包到本地仓库
deploy 部署到远程仓库

2. Clean 生命周期：

  • pre-clean - 清理前的准备工作
  • clean - 清理构建产生的文件
  • post-clean - 清理后的收尾工作

3. Site 生命周期：

  • pre-site - 生成站点前的准备工作
  • site - 生成项目站点文档
  • post-site - 生成站点后的收尾工作
  • site-deploy - 部署站点到服务器

4.2 常用命令

基本构建命令：

# 清理项目
mvn clean

# 编译项目
mvn compile

# 编译测试代码
mvn test-compile

# 运行测试
mvn test

# 打包项目
mvn package

# 安装到本地仓库
mvn install

# 部署到远程仓库
mvn deploy

组合命令：

# 清理并编译
mvn clean compile

# 清理、编译、测试、打包
mvn clean package

# 清理并安装到本地仓库
mvn clean install

# 跳过测试打包
mvn package -DskipTests

# 跳过测试编译和执行
mvn package -Dmaven.test.skip=true

依赖相关命令：

# 查看依赖树
mvn dependency:tree

# 分析依赖
mvn dependency:analyze

# 解决依赖
mvn dependency:resolve

# 下载源码
mvn dependency:sources

# 复制依赖到目录
mvn dependency:copy-dependencies

其他有用命令：

# 查看有效 POM
mvn help:effective-pom

# 查看有效设置
mvn help:effective-settings

# 查看插件信息
mvn help:describe -Dplugin=compiler

# 生成项目报告
mvn site

# 运行 Spring Boot 应用
mvn spring-boot:run

5.2 Profile 配置

Profile 允许为不同环境定制构建配置。

在 POM 中定义 Profile：

<profiles>
    <!-- 开发环境 -->
    <profile>
        <id>dev</id>
        <activation>
            <activeByDefault>true</activeByDefault>
        </activation>
        <properties>
            <env>dev</env>
            <database.url>jdbc:mysql://localhost:3306/myapp_dev</database.url>
            <log.level>DEBUG</log.level>
        </properties>
    </profile>
    
    <!-- 测试环境 -->
    <profile>
        <id>test</id>
        <properties>
            <env>test</env>
            <database.url>jdbc:mysql://test-server:3306/myapp_test</database.url>
            <log.level>INFO</log.level>
        </properties>
    </profile>
    
    <!-- 生产环境 -->
    <profile>
        <id>prod</id>
        <properties>
            <env>prod</env>
            <database.url>jdbc:mysql://prod-server:3306/myapp</database.url>
            <log.level>WARN</log.level>
        </properties>
        <build>
            <plugins>
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-compiler-plugin</artifactId>
                    <configuration>
                        <debug>false</debug>
                        <optimize>true</optimize>
                    </configuration>
                </plugin>
            </plugins>
        </build>
    </profile>
</profiles>

激活 Profile：

# 通过命令行激活
mvn clean package -Ptest

# 激活多个 Profile
mvn clean package -Ptest,integration

# 查看当前激活的 Profile
mvn help:active-profiles

Profile 激活条件：

<profile>
    <id>windows</id>
    <activation>
        <!-- 根据操作系统激活 -->
        <os>
            <family>windows</family>
        </os>
        <!-- 根据 JDK 版本激活 -->
        <jdk>11</jdk>
        <!-- 根据系统属性激活 -->
        <property>
            <name>environment</name>
            <value>test</value>
        </property>
        <!-- 根据文件存在激活 -->
        <file>
            <exists>src/main/config/app.properties</exists>
        </file>
    </activation>
</profile>

六、私有仓库配置

6.1 私有仓库配置概述

私有仓库用于存储企业内部的构件，提供更好的安全性和控制。

私有仓库的优势：

  • 安全性：内部构件不会暴露给外部
  • 稳定性：不依赖外部网络和服务
  • 控制性：可以控制依赖版本和更新
  • 性能：内网访问速度更快
  • 合规性：满足企业安全合规要求

常见私有仓库解决方案：

解决方案 类型 特点
Nexus Repository 商业/开源 功能强大，支持多种格式
Artifactory 商业 企业级功能丰富
Archiva 开源 Apache 项目，轻量级
GitLab Package Registry 集成 与 GitLab 集成

6.2 Nexus Repository 搭建

Docker 方式安装 Nexus：

# 创建数据目录
mkdir -p /opt/nexus-data
chown -R 200 /opt/nexus-data

# 运行 Nexus 容器
docker run -d \
  --name nexus \
  -p 8081:8081 \
  -v /opt/nexus-data:/nexus-data \
  sonatype/nexus3:latest

# 查看初始密码
docker exec nexus cat /nexus-data/admin.password

访问和初始配置：

  1. 访问 http://localhost:8081
  2. 使用 admin 用户和初始密码登录
  3. 完成设置向导，修改密码
  4. 配置匿名访问权限

创建仓库：

在 Nexus 管理界面中创建以下类型的仓库：

  • Hosted - 存储内部构件
  • Proxy - 代理外部仓库
  • Group - 组合多个仓库

创建 Maven Hosted 仓库：

  1. Repository → Repositories → Create repository
  2. 选择 maven2 (hosted)
  3. 配置仓库信息：
    • Name: maven-releases
    • Version policy: Release
    • Layout policy: Strict
  4. 同样创建 maven-snapshots 仓库 (Version policy: Snapshot)

创建 Maven Proxy 仓库：

# 代理中央仓库
Name: maven-central-proxy
Remote storage: https://repo1.maven.org/maven2/

# 代理阿里云仓库
Name: maven-aliyun-proxy
Remote storage: https://maven.aliyun.com/repository/public

创建 Maven Group 仓库：

Name: maven-public
Member repositories:
  - maven-releases
  - maven-snapshots
  - maven-central-proxy
  - maven-aliyun-proxy

6.3 构件发布配置

配置 settings.xml：

在 ~/.m2/settings.xml 中配置服务器认证信息：

<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0 
          http://maven.apache.org/xsd/settings-1.0.0.xsd">
    
    <!-- 服务器认证配置 -->
    <servers>
        <server>
            <id>nexus-releases</id>
            <username>admin</username>
            <password>your-password</password>
        </server>
        <server>
            <id>nexus-snapshots</id>
            <username>admin</username>
            <password>your-password</password>
        </server>
    </servers>
    
    <!-- 镜像配置 -->
    <mirrors>
        <mirror>
            <id>nexus-public</id>
            <mirrorOf>*</mirrorOf>
            <url>http://localhost:8081/repository/maven-public/</url>
        </mirror>
    </mirrors>
    
    <!-- Profile 配置 -->
    <profiles>
        <profile>
            <id>nexus</id>
            <repositories>
                <repository>
                    <id>nexus-public</id>
                    <url>http://localhost:8081/repository/maven-public/</url>
                    <releases><enabled>true</enabled></releases>
                    <snapshots><enabled>true</enabled></snapshots>
                </repository>
            </repositories>
            <pluginRepositories>
                <pluginRepository>
                    <id>nexus-public</id>
                    <url>http://localhost:8081/repository/maven-public/</url>
                    <releases><enabled>true</enabled></releases>
                    <snapshots><enabled>true</enabled></snapshots>
                </pluginRepository>
            </pluginRepositories>
        </profile>
    </profiles>
    
    <!-- 激活 Profile -->
    <activeProfiles>
        <activeProfile>nexus</activeProfile>
    </activeProfiles>
    
</settings>

配置项目 POM 发布：

<!-- 在项目 pom.xml 中配置分发管理 -->
<distributionManagement>
    <repository>
        <id>nexus-releases</id>
        <name>Nexus Release Repository</name>
        <url>http://localhost:8081/repository/maven-releases/</url>
    </repository>
    <snapshotRepository>
        <id>nexus-snapshots</id>
        <name>Nexus Snapshot Repository</name>
        <url>http://localhost:8081/repository/maven-snapshots/</url>
    </snapshotRepository>
</distributionManagement>

发布构件：

# 发布 SNAPSHOT 版本
mvn clean deploy

# 发布 Release 版本 (确保版本号不包含 SNAPSHOT)
mvn versions:set -DnewVersion=1.0.0
mvn clean deploy

# 使用 release 插件发布
mvn release:prepare
mvn release:perform

配置 Maven Release 插件：

<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-release-plugin</artifactId>
    <version>3.0.0</version>
    <configuration>
        <tagNameFormat>v@{project.version}</tagNameFormat>
        <autoVersionSubmodules>true</autoVersionSubmodules>
        <releaseProfiles>release</releaseProfiles>
        <goals>deploy</goals>
    </configuration>
</plugin>

七、最佳实践

7.1 Maven 最佳实践

项目结构最佳实践：

  • 遵循标准目录结构
  • 合理组织多模块项目
  • 使用有意义的 groupId 和 artifactId
  • 版本号遵循语义化版本规范

依赖管理最佳实践：

  • 使用 dependencyManagement 统一管理版本
  • 避免依赖冲突，合理使用 exclusions
  • 定期更新依赖版本
  • 使用 scope 合理控制依赖范围
  • 避免使用 SNAPSHOT 依赖在生产环境

构建配置最佳实践：

<!-- 统一编码和 JDK 版本 -->
<properties>
    <maven.compiler.source>11</maven.compiler.source>
    <maven.compiler.target>11</maven.compiler.target>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
</properties>

<!-- 锁定插件版本 -->
<build>
    <pluginManagement>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <version>3.11.0</version>
            </plugin>
        </plugins>
    </pluginManagement>
</build>

多模块项目结构：

parent-project/
├── pom.xml                    # 父 POM
├── common-module/             # 公共模块
│   └── pom.xml
├── service-module/            # 服务模块
│   └── pom.xml
├── web-module/                # Web 模块
│   └── pom.xml
└── integration-tests/         # 集成测试
    └── pom.xml

父 POM 配置示例：

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    
    <modelVersion>4.0.0</modelVersion>
    
    <groupId>com.example</groupId>
    <artifactId>parent-project</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>
    
    <modules>
        <module>common-module</module>
        <module>service-module</module>
        <module>web-module</module>
    </modules>
    
    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <spring.boot.version>2.7.0</spring.boot.version>
        <junit.version>5.8.2</junit.version>
    </properties>
    
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>${spring.boot.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>
    
</project>

安全最佳实践：

安全注意事项：

  • 不要在 POM 文件中硬编码密码
  • 使用 settings.xml 存储敏感信息
  • 考虑使用 Maven 密码加密
  • 定期扫描依赖漏洞
  • 使用 HTTPS 连接仓库

密码加密：

# 生成主密码
mvn --encrypt-master-password your-master-password

# 加密服务器密码
mvn --encrypt-password your-server-password

依赖漏洞扫描：

<!-- OWASP 依赖检查插件 -->
<plugin>
    <groupId>org.owasp</groupId>
    <artifactId>dependency-check-maven</artifactId>
    <version>8.2.1</version>
    <executions>
        <execution>
            <goals>
                <goal>check</goal>
            </goals>
        </execution>
    </executions>
</plugin>

# 运行漏洞扫描
mvn dependency-check:check

性能优化建议：

  • 使用本地仓库镜像
  • 合理配置并行构建
  • 使用增量编译
  • 优化测试执行
  • 使用构建缓存
